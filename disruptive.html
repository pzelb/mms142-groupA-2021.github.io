DIFFERENCE OF EDGE COMPUTING AND CLOUD COMPUTING

CLOUD COMPUTING

Cloud computing in the most simplest form is any computing processes that are done via the internet. All the data are accessed, stored, and processed through the web which are all located in a massive storage system of data called data centers.

EDGE COMPUTING

Edge computing on the other hand, can be seen as sort of the "middleman" that places all the computing resources closer to the user instead of having to travel from the main server and back which can cause latency issues. It is like storing it in a cache memory for later use. Edge computing is a game-breaking concept that when it is implemented alongside what we have now which is cloud computing, can solve a lot of security risks, delays, and better user experience.

Image from: Bosch.IO (2021)

Edge is more of an extension to cloud rather than a competition

“By processing incoming data at the edge, less information needs to be sent to the cloud and back. This also significantly reduces processing latency. A good analogy would be a popular pizza restaurant that opens smaller branches in more neighborhoods, since a pie baked at the main location would get cold on its way to a distant customer.” - Michael Clegg

HOW EDGE COMPUTING ENABLES INDUSTRY 4.0

Industry 4.0 (The Fourth Industrial Evolution)(4IR) is a breakthrough for technology as it is comprised of inventions that push the limits of the 3d world and the digital world. It is where technology makes use of data and smart machinery to improve and alter the human experience and livelihood. However, there are certain limitations that Industry 4.0 faces. Mainly it involves speed, versatility, and tech resources. This is where Edge Computing initiates the proper transition to the 4IR.

With its following aspects:

  • Interoperability - the capability to work with other systems now and in the future, future-proofed.
  • Granularity - The more granular the data, the more detailed it is and the more precise analysis can be.
  • Inclusive - equipped with both physical and informational identities.

Edge computing is an essential form of computer processing that can push us through the 4th Industrial Evolution.

CURRENT STATUS OF CLOUD SERVERS AND EDGE COMPUTING

Cloud adoption grew significantly in 2020 because of COVID-19. Plenty of the enterprises surveyed by Flexera (2021) have been adopting the multi-cloud approach as their dominant strategy to maximize the use of both multiple public and multiple private clouds. Despite the complexity and dynamic nature of the multi-cloud environment, public cloud spend has been increasing because of the demand of organizations. This may be attributed to the cloud's scalability, economy and reach.

Since edge computing is IoT-oriented, developers have been looking for sustainable ways of coping with the predicted volume of data IoT devices will be producing in the near future (Atos, 2021). The prevailing architecture of IoT is still centralized, so furthering the improvement of connectivity and response times through better edge data handling and security, edge software frameworks, function control and standardization are currently in the works.

Incorporating intelligence to edge is believed to heighten the security of edge devices and applications because its control can be multi-layered. It can cater to physical or perimeter security, firmware and operating systems, edge applications and edge networks. *Edge software frameworks are meant to optimize the heterogeneity of edge computing, the feature that allows edge devices to be hard-wired or battery-powered. The emphasis of the developments are on lifecycle management and real-time control functions. Developers are making the most out of the open source initiatives EdgeX Foundry, Eclipse Kura, KubeEdge and Open Stack Starlingx. *Ensuring the operational nature of edge environments maintains its value in the service sector and data quality. But since there are various challenges under service management, "multiple software solutions are often required to address each function and problem" (Atos, 2021). Thus, the need for a complete service ecosystem for edge management, namely: device managers, zero-touch provisioning, connectivity management platforms, and monitoring tools.

How is Edge Computing considered disruptive?

Edge computing can be traced back to the 1990s to Akamai's content delivery network (CDN). The idea back then was to place nodes closer to the end-user geographically for the transmission of cached data such as images and movies. It introduces a system that provides low-end to new customers.

Edge computing is a disruptive technology because it merges information technology (IT) and operational technology (OT) to match the growth of cloud computing, which generates a significant amount of data and high latencies. Edge computing accelerates acquiring, storing, and processing data because it places OT's data aggregation and IT's early analytics and computing next to each other. It essentially pulls specific functions of both technologies together to ease the back and forth flow of data and control. The closer data processing and decision-making are, the faster and more secure the flow becomes. In other words, edge computing doesn't exist to replace cloud computing; its purpose is to simply improve the performance and lower the cost of the cloud. The catch is the increasing demand for IoT devices, which do not need the cloud's centralized data storage feature to operate. Edge computing appeals to IoT because the former's decentralized infrastructure enables real-time computing—a function cloud computing does not offer.

In the simplest way, it is disruptive as it is currently under research and many beta testing. That said, it will still take a lot of time until edge computing is set for public use.
